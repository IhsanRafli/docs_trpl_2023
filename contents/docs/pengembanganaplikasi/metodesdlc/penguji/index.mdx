## Pengujian Fungsional 
Tujuan: Memastikan semua fungsi utama aplikasi berjalan sesuai kebutuhan.

### Halaman pengguna

memastikan halaman login menyimpan data pengguna dengan baik dan mencoba memasukkan data ke register agar terlihat apa berfungsi dengan normal atau ada terjadi kesalahan
dan mencobah mengakses dengan halaman logout dapat berfungsi dengan mengeluarkan akun dari sistem atau website dan mencobah uji testing dengan tombol button berfungsi atau tidaknya.
uji testing dengan fitur pencarian produk  dengan mencobah mencari produk dengan berdasarkan katagori dan nama produk. menguji bagian card atau tampilan daftar produk apakah button dapat mengirimkan data produk ke halaman cart jika user menambahkan produk ke halaman keranjang Belanja

### Halaman admin 

Menguji fitur crud dengan mencobah menambahkan produk ke halaman produk, uji button edit dengan data barang yang telah di tambahkan ke produk dan melakukan uji testing menghapus data produk dari halaman admin, menguji bagian button agar memastikan berfungsi sesuai dengan alur yang sudah ditentukan

## Pengujian Kegunaan (Usability Testing)
Tujuan: Memastikan aplikasi mudah digunakan dan memberikan pengalaman pengguna yang baik.

### Fokus Pengujian:
Kesederhanaan navigasi dan struktur menu.
Konsistensi antarmuka pengguna (warna, font, ikon).
Efektivitas alur pengguna, seperti pencarian produk dan checkout.
Feedback visual seperti loading spinner atau konfirmasi tindakan.

### Pengujian Responsif 
Tujuan: Memastikan aplikasi dapat diakses di berbagai perangkat dan resolusi layar.

### Fokus Pengujian:
Tampilan antarmuka pada perangkat desktop, tablet, dan ponsel.
Pengujian kompatibilitas di berbagai browser (Chrome, Safari, Firefox, Edge).
Responsivitas elemen seperti menu dropdown dan tombol.
