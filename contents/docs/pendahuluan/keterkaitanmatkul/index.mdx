## Kaitan Aplikasi dengan Mata Kuliah
1. Basis Data:
Dalam aplikasi ini, basis data menjadi komponen utama untuk menyimpan dan mengelola data pengguna, produk, transaksi, serta riwayat aktivitas.

- MySQL digunakan sebagai sistem manajemen basis data relasional (RDBMS), dengan struktur tabel yang dirancang untuk memastikan integritas data.
- Operasi seperti CRUD (Create, Read, Update, Delete) digunakan untuk memanipulasi data dengan efisien, menerapkan prinsip normalisasi untuk menghindari redundansi.
- Basis data ini juga dirancang untuk mendukung fitur pencarian cepat dan transaksi yang aman menggunakan query SQL yang teroptimasi.


2. Metode dan Model Pengembangan Perangkat Lunak (SDLC Waterfall):
Dalam pengembangan aplikasi ini, kami menggunakan model Waterfall, yang berfokus pada penyelesaian setiap tahap secara berurutan.

- Tahap 1: Analisis Kebutuhan:
Mengidentifikasi kebutuhan sistem dengan mendetail, termasuk fitur utama seperti manajemen produk, transaksi, dan integrasi pembayaran.
- Tahap 2: Desain Sistem:
Membuat blueprint aplikasi yang mencakup desain basis data menggunakan ERD (Entity-Relationship Diagram) dan desain proses menggunakan Use Case Diagram serta Sequence Diagram.
- Tahap 3: Implementasi:
Melibatkan pengkodean aplikasi menggunakan HTML, CSS, dan JavaScript untuk frontend, serta PHP untuk backend. Setiap modul dikembangkan sesuai desain yang telah dirancang.
- Tahap 4: Pengujian:
Aplikasi diuji untuk memastikan semua fitur berfungsi dengan baik, termasuk pengujian fungsionalitas basis data, integrasi antar modul, dan performa sistem.
- Tahap 5: Deployment:
Aplikasi diimplementasikan di server produksi dan diuji ulang untuk memastikan kompatibilitas di lingkungan nyata.
- Tahap 6: Pemeliharaan:
Setelah aplikasi berjalan, dilakukan pemantauan dan perbaikan terhadap bug atau kebutuhan tambahan dari pengguna.


3. Perancangan Antarmuka Pengguna:
Perancangan antarmuka pengguna dilakukan dengan prinsip User-Centered Design (UCD) untuk memastikan kemudahan penggunaan bagi semua kalangan.

- Menggunakan tools desain seperti Figma untuk prototipe.
- Penerapan prinsip Konsistensi dan Standar serta Hirarki Visual dalam desain antarmuka, sehingga elemen seperti navigasi, tombol, dan formulir mudah dipahami pengguna.
- Pengujian antarmuka dilakukan untuk mengukur usability dan memastikan aplikasi memberikan pengalaman pengguna yang optimal.


4. Pemrograman Web Lanjut:

- Aplikasi ini mengintegrasikan HTML, CSS, dan JavaScript pada frontend untuk menciptakan halaman web yang dinamis dan responsif.
- Backend dibangun menggunakan PHP, yang mendukung pengelolaan data secara real-time dan proses server-side yang aman.
- Fitur-fitur tambahan seperti API untuk integrasi payment gateway dan validasi data menggunakan JavaScript melibatkan konsep pemrograman web tingkat lanjut.


5. Praktik Analisis dan Desain Perangkat Lunak:

- Analisis Kebutuhan: Menggunakan diagram UML seperti Use Case Diagram untuk memetakan kebutuhan pengguna, baik pembeli maupun penjual.
- Desain Sistem: Dirancang menggunakan ERD (Entity-Relationship Diagram) untuk menggambarkan struktur basis data serta Sequence Diagram untuk menjelaskan alur proses utama seperti pembelian produk.
